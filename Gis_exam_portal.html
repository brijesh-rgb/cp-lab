Q1
What is GIS? Explain Key Components and Application Areas
202220232024 6 Marks Unit 1 ⭐ High Freq

GIS is a computer-based system to capture, store, analyze, manage, and display geographically referenced data. 5 components: Hardware, Software, Data (Spatial + Attribute), People, Methods. Applications: Urban planning, Disaster management, Agriculture, Transport, Environment.

Introduction

A Geographic Information System (GIS) is an integrated computer-based system designed to capture, store, manipulate, analyze, manage, and visualize all types of geographically referenced data. GIS links spatial data (WHERE — geometry, location) with attribute data (WHAT — characteristics, properties) to support decision-making across diverse domains.

Five Key Components of GIS

1. Hardware
  • Computers: Workstations, servers for geospatial processing
  • Input: GPS receivers, scanners, digitizing tablets, remote sensors
  • Output: High-res monitors, plotters, printers
2. Software
  • GIS tools: ArcGIS, QGIS, MapInfo, Google Earth Engine
  • Functions: Data input, spatial analysis, visualization, database management
3. Data (Most Critical)
  • Spatial Data: Geometry — points, lines, polygons, rasters
  • Attribute Data: Characteristics — population, land use, road name
  • Metadata: Data about data — source, accuracy, coordinate system
4. People
  • GIS analysts, cartographers, database administrators
  • End users: Urban planners, environmentalists, government officials
5. Methods/Procedures
  • Standardized workflows for data collection, georeferencing, and analysis
  • Quality assurance protocols, digitization standards, SOPs

🌍 Real-World Example

During the 2018 Kerala Floods, GIS was used to: map affected villages (raster satellite data), identify evacuation routes (vector road networks), coordinate relief distribution using population density layers, and predict further flood zones using Digital Elevation Models (DEM).

GIS is a powerful multi-disciplinary platform that transforms raw geographic data into actionable intelligence. All five components must work in harmony. It is indispensable in modern governance, environmental science, and urban development.

Q2
Compare Raster vs Vector Data Models – Advantages & Disadvantages
202220232024 6 Marks Unit 1 & 2 ⭐ High Freq

Vector: Points/Lines/Polygons, precise coordinates, small size, good for discrete features (roads, boundaries). Raster: Grid of pixels, each cell = one value, large size, good for continuous data (elevation, satellite imagery). Key difference: Vector is exact; Raster is cell-approximated.

Introduction

GIS uses two fundamental data models to represent the real world. Vector data represents features using geometric primitives. Raster data divides space into a regular grid of cells. Both are complementary — modern GIS uses them together.

Advantages & Disadvantages
Vector — Advantages
  • Precise boundary representation with exact coordinates
  • Compact file size — only store feature vertices
  • Supports topology relationships (adjacency, connectivity)
  • Easy attribute linkage to each discrete feature
Vector — Disadvantages
  • Complex data structure — topology rules required
  • Not ideal for continuous surface data (elevation, temperature)
  • Computationally heavy overlay operations
Raster — Advantages
  • Simple matrix structure — easy to implement
  • Efficient for surface analysis (slope, viewshed, DEM)
  • Easy overlay — cell-by-cell operations
  • Compatible with remote sensing data (satellite images)
Raster — Disadvantages
  • Large file sizes at high resolution
  • Loss of precision at feature boundaries (mixed pixel problem)
  • All cells must be same size — no variable density
🌍 Real-World Example: Smart City GIS – Pune
  • Vector: Road network (lines), building footprints (polygons), bus stops (points)
  • Raster: Satellite image, elevation model, land surface temperature map
  • Combined analysis identifies optimal locations for new bus stops

Vector and Raster are complementary, not competing. Modern GIS integrates both — Vector for discrete features needing precise boundaries, Raster for continuous phenomena like elevation and temperature. The choice depends on the nature of geographic data and the type of analysis required.

Q3
TIN (Triangulated Irregular Network) Data Model – Terminologies & Comparison
202220232024 6 Marks Unit 1

TIN = Vector-based 3D surface model using irregularly spaced triangles. Key terms: Node (x,y,z point), Edge (triangle side), Face (triangle surface), Breakline (ridges/valleys), Delaunay triangulation. Advantage over raster DEM: Variable density — more detail in complex terrain.

Introduction

A Triangulated Irregular Network (TIN) is a vector-based surface representation model used to model terrain in 3D. TIN divides geographic space into a network of non-overlapping contiguous triangles formed from irregularly spaced sample points, each carrying x (longitude), y (latitude), and z (elevation) coordinates.

📐 TIN Structure Diagram
P1(x1,y1,150m) P4(x4,y4,200m) *─────────────────────────* | ╲ T2 ╱ | | ╲ ╱ | | T1 ╲ ╱ T4 | | ╲ T3 ╱ | | P3(180m) | *──────────*──────────────* P2(120m) P5(100m) P = Node(x,y,z) T = Triangle Face Lines = Edges
Fig: TIN with nodes (elevation points), edges, and triangular faces
Key Terminologies
  • Node/Vertex: Sample point with known (x, y, z) — the building block of TIN
  • Edge: Line connecting two adjacent nodes forming triangle side
  • Face/Triangle: 3D planar surface formed by 3 connected nodes
  • Delaunay Triangulation: Algorithm to form triangles — maximizes minimum angles to avoid slivers
  • Voronoi Diagram: Dual graph of TIN — areas closest to each node
  • Breaklines: Force triangle edges along ridges, valleys, or cliff lines for accuracy
  • Mass Points: Additional elevation points added to improve surface accuracy
  • Hull: Outer boundary of the entire TIN network

🌍 Real-World Example: Flood Modeling

Civil engineers use TIN to model river valleys — breaklines along banks give precise channel shape; variable triangle density provides detail near steep banks and fewer triangles in flat floodplains, making flood simulation more accurate than raster DEM.

TIN is the preferred model for terrain analysis when accuracy matters, especially for hydrology, civil engineering grade estimation, and landslide modeling. Its variable density makes it both accurate and storage-efficient for complex 3D surfaces.

Q4
Raster Data Compression Techniques – Chain Coding & Block Encoding
202220232024 6 Marks Unit 2 ⭐ High Freq

Chain Coding: Encodes boundary direction using 8-directional codes (0–7). Stores starting point + direction sequence. Good for boundary shapes. Block Encoding: Finds largest uniform square blocks in raster; stores (row, col, size, value). Good for large uniform areas. Both are lossless.

1. Chain Coding (Freeman Chain Code)

Chain coding compresses raster data by encoding the boundary of a region using directional codes. Instead of storing every cell coordinate, it traces the perimeter using 8-directional step codes.

📐 Freeman Chain Code Direction System
8-Directional Freeman Chain Code: 3 2 1 \ | / 4 ── * ── 0 (0=East, 1=NE, 2=North, 3=NW, / | \ 4=West, 5=SW, 6=South, 7=SE) 5 6 7 Example boundary trace → right→up→right→down: Chain Code: 0, 2, 0, 6 (4 values vs 8 coords!) Draw: Polygon on grid with arrows showing direction codes
Fig: 8-directional chain code; each step = 1 code instead of 2 coordinates
Process
  • Step 1: Identify starting cell on boundary of region
  • Step 2: Trace boundary clockwise, record direction code at each step
  • Step 3: Store: starting point coordinates + sequence of direction codes
  • Step 4: Decode by following direction codes from start
Advantages & Disadvantages
  • ✅ Efficient for compact shapes with smooth boundaries
  • ✅ Preserves shape information with minimal data
  • ❌ Only encodes boundaries, not interior fill
  • ❌ Less efficient for highly irregular boundaries
2. Block Encoding

Block encoding divides the raster into square blocks of uniform value and stores block position, size, and value instead of individual cells. It is a 2D extension of Run-Length Encoding (RLE).

📐 Block Encoding Example
Original 5×5 Raster: Block Encoded: ┌──┬──┬──┬──┬──┐ Block 1: (1,1, size=3, val=A) │A │A │A │B │B │ Block 2: (1,4, size=2, val=B) ├──┼──┼──┼──┼──┤ Block 3: (4,1, size=2, val=C) │A │A │A │B │B │ Block 4: (4,3, size=1, val=D) ├──┼──┼──┼──┼──┤ │A │A │A │D │D │ Original: 25 values stored ├──┼──┼──┼──┼──┤ Encoded: 6 records only! │C │C │D │D │D │ └──┴──┴──┴──┴──┘ Draw: Color-coded grid showing uniform blocks with dashed borders
Fig: Large uniform blocks stored as single records — significant compression
Process
  • Step 1: Scan raster from top-left to bottom-right
  • Step 2: Identify largest possible square block of uniform value
  • Step 3: Record: (Row, Column, Block Size, Value)
  • Step 4: Advance to next unprocessed area and repeat
Advantages & Disadvantages
  • ✅ Highly effective for large uniform areas (ocean, desert, forest)
  • ✅ Simple to implement and decode
  • ❌ Less effective for heterogeneous rasters (many small patches)

🌍 Real-World Example: ISRO Satellite Imagery

Block encoding compresses large uniform land cover classes (ocean=blue, desert=yellow, forest=green) reducing a 500MB raster to ~50MB. Chain coding precisely traces coastlines and administrative boundaries. Used together in national-scale GIS databases.

Chain Coding and Block Encoding are complementary lossless compression methods. Chain coding excels at boundary representation; block encoding efficiently compresses homogeneous regions. Both reduce storage without any data loss, making them critical for large raster GIS datasets.

Q5
Non-Spatial Data Attributes – Types: Nominal, Ordinal, Interval, Ratio
20232024 6 Marks Unit 2

4 Types: (1) Nominal – categories, no order (land type = residential/commercial). (2) Ordinal – ranked but unequal intervals (risk = low/medium/high). (3) Interval – equal intervals, no true zero (temperature °C). (4) Ratio – equal intervals + true zero (area m², population). Hierarchy: Nominal → Ordinal → Interval → Ratio.

Introduction

Non-spatial attribute data in GIS describes WHAT geographic features are — their characteristics, measurements, and properties. Stored in relational tables linked to spatial geometry via unique IDs. Classified into 4 measurement scales with increasing analytical power.

Four Types of Attribute Data
1. Nominal (Categorical)
  • Categories/names with no inherent order or magnitude
  • Only classifies features into distinct groups
  • GIS Examples: Land use type (residential/commercial/agricultural), soil type, road classification (highway/local/rural)
  • Analysis: Count frequency, find mode, query by category
  • Map type: Qualitative colors — each category has distinct hue
2. Ordinal (Ranked)
  • Places features in meaningful order, but intervals between ranks are NOT equal
  • GIS Examples: Road quality (Excellent/Good/Fair/Poor), risk level (Low/Medium/High), stream order (1st, 2nd, 3rd)
  • Analysis: Rank, sort, compare relative position
  • Map type: Sequential color scheme (light → dark)
3. Interval
  • Equal intervals between values, but no true zero (zero is arbitrary)
  • Ratios are meaningless: 40°C is NOT twice as hot as 20°C
  • GIS Examples: Temperature in °C, calendar year, elevation from arbitrary datum
  • Analysis: Addition, subtraction valid; multiplication/division NOT
  • Map type: Diverging color scheme (e.g., blue-white-red for temperature)
4. Ratio (Highest Level)
  • Equal intervals AND true zero — zero = absence of the quantity
  • All mathematical operations valid (including ratios and percentages)
  • GIS Examples: Population count, area in m², distance, rainfall in mm, income, crop yield
  • Analysis: Mean, standard deviation, ratios, percentages — all valid
  • Map type: Proportional symbols or graduated colors
Hierarchy & Summary
NOMINAL → ORDINAL → INTERVAL → RATIO (Categories) → (Ranked) → (Equal Intervals) → (True Zero) Each level inherits ALL properties of levels below it

🌍 Real-World Example: PMRDA Pune Land Parcel GIS
  • Nominal: Zone type = Residential / Commercial / Industrial / Green
  • Ordinal: Development priority = High / Medium / Low
  • Interval: Year of construction = 1990, 2005, 2018
  • Ratio: Plot area = 500 m², 1200 m² (half plot = half area — meaningful)

Understanding attribute data types is fundamental to correct GIS analysis. Selecting the wrong analysis method for a data type (e.g., calculating mean of nominal data) produces meaningless results. The four-level hierarchy guides which statistical operations are valid.

Q6
Spatial Database Systems in GIS – Types and Functionalities
20222023 6 Marks Unit 2

Spatial Database System (SDBS) = DBMS extended to handle geometry + attributes. Types: (1) Object-Relational: PostGIS, Oracle Spatial (most common); (2) Object-Oriented: GemStone; (3) NoSQL: MongoDB with GeoJSON; (4) File-based: Shapefile, GeoPackage; (5) Raster DB: PostGIS Raster. Key feature: Spatial indexing (R-Tree) + SQL spatial functions.

Introduction

A Spatial Database System (SDBS) is a DBMS extended to store, manage, query, and analyze both spatial (geographic geometry) and non-spatial (attribute) data. Unlike conventional databases, spatial DBs understand geometry, location, and spatial relationships.

Types of Spatial Database Systems
1. Object-Relational SDBS (Most Common)
  • Extends relational DBMS with spatial data types and functions
  • Examples: PostGIS (PostgreSQL), Oracle Spatial, SQL Server Spatial, IBM DB2 Spatial
  • Uses SQL with spatial extensions (ISO SQL/MM standard)
  • Spatial functions: ST_Distance(), ST_Buffer(), ST_Intersects(), ST_Union()
2. Object-Oriented SDBS
  • Stores geographic features as objects with attributes AND methods
  • Examples: GemStone, ObjectStore
  • Best for complex 3D GIS and CAD/GIS integration
3. NoSQL Spatial Databases
  • MongoDB: Stores spatial data as GeoJSON documents
  • Apache Cassandra: For large-scale distributed IoT spatial data
  • Best for web-scale, high-volume real-time spatial applications
4. File-Based Spatial Storage
  • Shapefile (.shp): Most common simple format — ESRI standard
  • GeoPackage (.gpkg): SQLite-based, OGC open standard
  • Geodatabase (.gdb): ESRI proprietary — complex relationships
5. Raster Databases
  • PostGIS Raster, Oracle GeoRaster, ESRI Raster Catalog
  • Stores satellite imagery and DEMs alongside attribute data
🌍 Real-World Example: BBMP Bangalore Smart City GIS

Uses PostGIS (PostgreSQL + PostGIS extension). Query: "SELECT roads WHERE ST_DWithin(geom, pothole_location, 500)" → finds roads within 500m of reported potholes for maintenance priority. Field staff update via mobile GIS apps connected to PostGIS server in real-time.

Object-Relational databases like PostGIS and Oracle Spatial are the most widely adopted, combining full SQL capability with robust spatial analysis. The best SDBS choice depends on scale, infrastructure, performance needs, and analytical requirements.

Q7
Map Projections in GIS – Types, Properties & Applications
202220232024 6 Marks Unit 3 ⭐ High Freq

Map projection = mathematical transformation of 3D Earth onto 2D flat plane. 3 surface types: Cylindrical (Mercator, UTM), Conical (Lambert, Albers), Azimuthal/Planar (Stereographic). 4 properties: Conformal (shape), Equal Area (area), Equidistant (distance), Azimuthal (direction). NO projection preserves all four simultaneously.

Introduction

Map projection is the mathematical transformation of Earth's curved 3D surface onto a flat 2D plane. Since Earth is a geoid/ellipsoid, ALL projections introduce distortions in shape, area, distance, or direction. No single projection preserves all four properties simultaneously.

By Property Preserved
  • Conformal (Orthomorphic): Preserves shape/angles — Mercator, Lambert Conformal Conic
  • Equal Area (Equivalent): Preserves area — Albers Equal Area, Sinusoidal
  • Equidistant: Preserves distance from center — Plate Carrée, Azimuthal Equidistant
  • Azimuthal: Preserves direction from center — Lambert Azimuthal Equal Area
Major GIS Projection Systems
1. Mercator (Cylindrical Conformal)
  • Preserves shape and angles (rhumb lines are straight)
  • Severely distorts area near poles (Greenland ≈ Africa in size)
  • Application: Navigation charts, Google Maps (Web Mercator)
2. UTM – Universal Transverse Mercator
  • Divides world into 60 north-south zones, each 6° wide
  • High accuracy for north-south oriented areas
  • Application: India topographic maps (UTM Zones 43–46N)
3. Lambert Conformal Conic
  • Preserves shape in mid-latitudes; two standard parallels
  • Application: Survey of India maps, aeronautical charts
4. Albers Equal Area Conic
  • Preserves area — suited for E-W extent countries
  • Application: National land use and census thematic maps
🌍 Real-World Examples
  • Google Maps: Web Mercator (EPSG:3857) — shapes correct, area distorted near poles
  • Survey of India: Lambert Conformal Conic (12°N and 32°N standard parallels)
  • ISRO Bhuvan: WGS84 Geographic Coordinate System
  • Land records (PMRDA): Albers Equal Area for accurate plot area measurement

Projection selection is critical in GIS. Choose based on: location (latitude of study area), purpose (navigation vs area calculation), and spatial extent. Every projection distorts something — know what's distorted and whether it matters for your analysis.

Q8
Spatial Interpolation Techniques – IDW and Kriging
20232024 6 Marks Unit 3 ⭐ High Freq

IDW: Deterministic; weight = 1/distance^p; no error estimation; fast; bull's eye effect; best for dense uniform data. Kriging: Geostatistical; uses variogram (Nugget, Sill, Range); provides error/uncertainty map; optimal unbiased estimator; best for spatially correlated data (mining, groundwater). Kriging > IDW in accuracy but more complex.

Introduction

Spatial interpolation estimates unknown values at unsampled locations from known sample points. Based on Tobler's First Law of Geography: "Nearby things are more similar than distant things." Two major techniques: IDW (deterministic) and Kriging (geostatistical).

1. Inverse Distance Weighting (IDW)

IDW estimates unknown values as a weighted average of known points, where weight is inversely proportional to distance raised to power p.

Z* = Σ(Zᵢ / dᵢᵖ) / Σ(1 / dᵢᵖ) Z* = Estimated value Zᵢ = Known value at point i dᵢ = Distance to point i p = Power (usually p = 2)
Key Characteristics
  • Deterministic: No statistical error estimation
  • Bull's Eye Effect: Values cluster around known points creating concentric rings
  • Power p: Higher p → nearby points dominate more; lower p → smoother surface
  • When to use: Dense, evenly distributed sample data (rainfall, temperature)
2. Kriging (Geostatistical Interpolation)

Kriging uses statistical models (variogram) to estimate unknown values AND provide uncertainty estimates. It considers both distance AND the spatial autocorrelation structure of data — making it the Best Linear Unbiased Estimator (BLUE).

📐 Variogram (Semivariogram) – Core of Kriging
VARIOGRAM: Semivariance | ─── Sill (max variability) | ───── | ───── | ───── | ───── |──── +──────────────────────── Distance (Lag) | | | Range (where autocorrelation ends) Nugget (error at zero distance) Nugget: measurement error Sill: max variability Range: autocorrelation limit
Fig: Semivariogram shows how spatial similarity decreases with distance
Types of Kriging
  • Ordinary Kriging: Most common; assumes stationary mean
  • Simple Kriging: Known constant mean across domain
  • Universal Kriging: Accounts for spatial trend
  • Co-Kriging: Uses multiple correlated variables
  • Indicator Kriging: For categorical data and probabilities
Kriging Process
  • Step 1: Compute experimental variogram from sample data
  • Step 2: Fit theoretical model (spherical/Gaussian/exponential) to variogram
  • Step 3: Use variogram model to compute optimal kriging weights
  • Step 4: Apply weights → estimated value + prediction variance (error map)

🌍 Real-World Example: Maharashtra
  • IDW: IMD interpolates rainfall from 200 weather stations — fast, works for dense stations
  • Kriging: Hydrogeology maps groundwater depth in Marathwada drought zone — provides uncertainty maps for risk assessment decisions

IDW is simpler and faster for dense, evenly distributed data. Kriging is statistically rigorous, providing optimal estimates with uncertainty quantification — preferred for scientific and precision applications like mining, hydrology, and environmental monitoring.

Q9
Digital Elevation Model (DEM) – Definition, Applications & Comparison with DSM/DTM
202220232024 6 Marks Unit 3 & 4 ⭐ High Freq

DEM = Raster grid where each cell = elevation above sea level (bare ground only). Sources: SRTM (30m), LiDAR (1m), ASTER. Applications: slope, watershed, flood mapping, viewshed, road planning. DEM = bare earth; DSM = includes buildings/trees; DTM = terrain + vector features.

Introduction

A Digital Elevation Model (DEM) is a digital raster representation of the Earth's bare-ground topographic surface. Each cell stores the elevation value in meters above mean sea level. DEMs are the foundational dataset for all terrain analysis in GIS and remote sensing.

📐 DEM Grid & 3D Terrain Visualization

Applications of DEM
  • Slope & Aspect Analysis: Terrain steepness and direction for erosion and land use planning
  • Watershed Delineation: Drainage basin boundaries for hydrological modeling
  • Flood Zone Mapping: Simulate inundation at various water levels
  • Road Planning: Optimal route design avoiding steep unstable terrain
  • Viewshed Analysis: Visible areas from telecom towers, military posts
  • Volume Calculations: Cut-and-fill for construction, dam capacity
  • Landslide Risk: Identify unstable steep slopes in mountainous terrain
  • Climate Modeling: Elevation-dependent temperature and rainfall patterns

DEM is the foundational dataset for terrain analysis. While DEM = bare earth, DSM includes above-ground features and DTM adds vector breaklines. SRTM remains the most widely used global DEM; LiDAR provides sub-meter accuracy where high precision is needed.

Q10
Quality Assessment of Freely Available DEMs
20222023 6 Marks Unit 4

Quality of DEMs assessed via: Vertical accuracy (RMSE against ground truth GPS), Horizontal accuracy, Spatial resolution (30m SRTM vs 1m LiDAR), Void coverage, Vegetation/building effects. Common DEMs: SRTM (RMSE ±16m), ASTER (RMSE ±10–25m), Cartosat (RMSE ±8m), LiDAR (<±0.15m). LiDAR = most accurate; SRTM = most accessible.

Introduction

Quality assessment of DEMs involves evaluating how accurately a DEM represents the actual terrain. Several freely available global DEMs exist, each with different accuracy levels. Quality is assessed through vertical accuracy, horizontal accuracy, resolution, and coverage completeness.

Quality Assessment Parameters
1. Vertical Accuracy (Most Important)
  • Measured using RMSE (Root Mean Square Error) against GPS ground control points
  • RMSE = √(Σ(DEM_elevation − GPS_elevation)² / n)
  • Lower RMSE = Higher accuracy
2. Horizontal (Positional) Accuracy
  • Accuracy of x,y location of terrain features in the DEM
  • Evaluated against high-precision GPS or optical imagery
3. Spatial Resolution
  • Cell size determines level of terrain detail captured
  • 30m SRTM cannot capture features smaller than 30m × 30m
4. Data Voids & Artifacts
  • Voids: Missing data over steep terrain or radar shadow areas (SRTM)
  • Artifacts: Striping noise, speckle, vegetation bias
  • Evaluated by: visual inspection, histogram analysis, cross-section profiles

  • GPS Ground Truth: Compare DEM elevation at known GPS points → compute RMSE
  • Cross-Section Profiles: Extract elevation transects and compare with field survey
  • Histogram Analysis: Detect data voids and outlier elevations
  • Contour Overlay: Derive contours from DEM and compare with official topo maps
  • Error Raster: Generate difference raster (DEM − reference) to visualize error distribution
🌍 Real-World: ISRO DEM Comparison Study

ISRO compared SRTM, ASTER, and Cartosat DEMs over the Western Ghats. Result: Cartosat DEM showed lowest RMSE (±8m) due to optical stereo pair data. SRTM had voids over steep valleys due to radar shadowing. LiDAR DEM used for high-priority micro-watershed studies showed RMSE < 0.5m.

No single freely available DEM is perfect for all applications. SRTM is the most accessible but has voids and ±16m vertical error. ALOS AW3D30 offers better accuracy for free. LiDAR is the gold standard but expensive. Quality assessment before use is mandatory for critical applications.

7 Steps: (1) Problem formulation → (2) Data collection → (3) Pre-processing (georef, clean) → (4) Data management (DB) → (5) Spatial analysis (overlay, buffer, network) → (6) Output (maps, reports) → (7) Interpretation/Decision. Analysis types: Overlay, Buffer, Network, Terrain, Statistical, Interpolation.

Step 1: Problem Formulation
  • Define the spatial question clearly: What decision needs to be made?
  • Specify study area boundary and time period
  • Example: "Find best location for hospital in Pune serving max population"
Step 2: Data Collection & Input
  • Sources: Satellite imagery, GPS field survey, census data, existing maps
  • Formats: Shapefile, GeoTIFF, KML, GeoJSON

  • Georeferencing: Assign real-world coordinates
  • Coordinate system: Transform all layers to common projection

GIS analysis is a systematic, iterative workflow transforming raw geographic data into actionable insights. Each step builds upon the previous. The quality of output depends entirely on the quality of input data and the appropriateness of analysis methods chosen.

Q12
Errors in GIS – Types, Causes and Impact on Analysis
202220232024 6 Marks Unit 5 ⭐ High Freq

GIS Error Types: (1) Source: outdated/incomplete data. (2) Positional: wrong coordinates. (3) Attribute: wrong values. (4) Topological: undershoot, overshoot, sliver polygon, dangling node. (5) Processing: projection, interpolation, rounding errors. (6) Raster: mixed pixel, georeferencing, classification error. Detection: topology rules, GPS validation, metadata review.

Introduction

GIS errors are discrepancies between real-world geographic features and their digital representation. They propagate through analysis operations and can lead to incorrect conclusions. Errors must be detected and minimized through rigorous quality control.

Types of GIS Errors
1. Source / Input Errors
  • Age/Currency: Outdated data not reflecting current reality
  • Coverage: Incomplete spatial coverage — some areas missing
  • Scale Error: Small-scale data lacks detail needed for large-scale analysis
  • Generalization: Features simplified or omitted during mapping
2. Positional (Geometric) Errors
  • Features placed at wrong geographic coordinates due to GPS error or poor digitizing
  • GPS accuracy: Standard GPS ±5m; DGPS ±2cm
3. Attribute Errors
  • Wrong attribute values assigned to spatially correct features
  • Example: Plot ownership incorrectly entered in cadastral GIS database
4. Topological Errors (Critical for Vector)
📐 Common Topology Errors

Fig: Common topology errors in vector GIS data
5. Processing Errors
  • Projection Error: Wrong coordinate transformation → layers misaligned by meters/km
  • Interpolation Error: Poor variogram → inaccurate surface estimates
  • Rounding Error: Loss of coordinate precision during computation
  • Overlay Error: Sliver polygons from imperfect boundary alignment
6. Raster-Specific Errors
  • Mixed Pixel Problem: Boundary cells contain multiple land cover types
  • Georeferencing Error: Raster misaligned with true ground coordinates
  • Classification Error: Remote sensing image classified into wrong land cover class

Detection Methods
  • Topology rules in ArcGIS: Check undershoots, overshoots, gaps
  • GPS ground truth validation: Compare DEM or vector data with field measurements
  • Metadata review: Check source, scale, coordinate system of input data
  • Visual inspection: Overlay with reference data to spot misalignment
🌍 Real-World: Land Title Disputes – Urban India

Positional errors in cadastral maps cause boundary disputes between adjacent plots. Sliver polygons from overlay of municipal and revenue boundaries create 'no man's land' zones with unclear jurisdiction. Solution: DGPS surveys with ±2cm accuracy for cadastral GIS correction.

GIS errors are inevitable but manageable. They originate from data sources, measurement processes, and analysis operations. Rigorous quality control — topology checks, metadata documentation, and ground truth validation — minimizes error propagation and ensures reliable GIS-based decision-making.

Q13
Key Elements of a Map (Map Furniture)
20222023 6 Marks Unit 5

9 essential map elements: Title (what and where), Scale (RF/bar/verbal), Legend/Key (symbol meaning), North Arrow (orientation), Graticule/Grid (coordinates), Inset Map (location context), Data Source & Date, Projection Info (CRS), Neat Line/Border. Without these, a map is ambiguous.

Introduction

A map must include specific cartographic elements called Map Furniture to be readable, accurate, and professionally usable. These allow any reader to correctly interpret spatial information without needing additional context.

📐 Standard Map Layout Template
┌─────────────────────────────────────────────────────┐ │ MAP TITLE [N↑] │ │──────────────────────────────────────────────────── │ │ ┌──────────────────────────┐ ┌────────────────┐ │ │ │ │ │ LEGEND │ │ │ │ MAIN MAP AREA │ │ ■ Forest │ │ │ │ (Geographic features, │ │ ~ River │ │ │ │ layers, symbology) │ │ ● City │ │ │ │ │ │ □ Boundary │ │ │ │ [Graticule/Grid lines] │ └────────────────┘ │ │ └──────────────────────────┘ ┌────────────────┐ │ │ Scale: |──── 1 km ────| │ INSET MAP │ │ │ Source: SOI 2023 │ [India map] │ │ │ Projection: WGS84 UTM 43N └────────────────┘ │ └─────────────────────────────────────────────────────┘
Fig: Standard map layout showing all key elements/furniture
Nine Key Map Elements
1. Title
  • Identifies subject and geographic area; placed prominently at top center
  • Example: "Land Use Map of Pune Municipal Corporation – 2024"
2. Scale
  • Representative Fraction (RF): 1:50,000 → 1 unit on map = 50,000 on ground
  • Graphic Scale Bar: Visual bar with ground distances — best for resized maps
  • Verbal Scale: "1 cm = 500 m"
3. Legend / Key
  • Explains ALL symbols, colors, patterns used on the map
  • Must be complete — every symbol must be explained
4. North Arrow / Compass Rose
  • Shows map orientation; essential for navigation and spatial reasoning
  • Simple N arrow or full decorative compass rose
5. Graticule / Grid Reference
  • Latitude/longitude lines (geographic) or UTM grid (projected coordinates)
  • Enables precise feature location referencing
6. Inset Map
  • Small overview map showing where the main map fits in larger context
  • Example: India outline with Maharashtra highlighted → Pune highlighted
7. Data Source & Date
  • Credits data sources; date crucial for assessing currency of information
  • Example: "Source: Survey of India, 2022 | Cartosat DEM 30m"
8. Projection Information
  • States coordinate reference system (CRS) used
  • Example: "WGS 84, UTM Zone 43N, EPSG:32643"
9. Neat Line / Border
  • Rectangular frame separating map content from surrounding map elements
🌍 Real-World: ISRO Bhuvan Maps
  • Title: "Vegetation Cover Map – India 2023"
  • Scale bar: 0–500–1000 km
  • Legend: NDVI color scale (dark red=dense to yellow=sparse)
  • Source: Resourcesat-2 LISS-III | Date: October 2023 | WGS84

All key map elements together make a map self-explanatory, accurate, and universally readable. Without these elements, a map becomes ambiguous or misleading. In professional GIS, map layout design is as important as the spatial analysis itself.
